自3年前看过两本关于Ruby的书外，基本没有再看Ruby的资料了，因这次有高人指点，又要重新学起Ruby了，他们解决了一个非常好的工具给我，RVM。这个工具可以虚拟一个指定版本的Ruby环境出来，就好象Python下的virtualenv差不多，不过操作起来不够virtualenv那么方便，但整体感觉还是不错的。 OK，由于CentOS的源是没有RVM的，所以需要download安装，但开发者已经为我们考虑周全了。所以可以通过一个简单的命令将问题解决。

[bash] # 如果没有安装curl sudo yum install curl curl -L get.rvm.io | bash -s stable [/bash] 安装完RVM后，加载RVM环境 [bash] # 我是用Root用户安装的，我的rvm环境配置文件会再/etc/profile.d下 source /etc/profile.d/rvm.sh [/bash] 接着安装RVM的依赖 [bash] yum install libyaml-devel -y rvm requirements [/bash] 依赖解决了之后继续安装一些通用的工具,使用rvmsudo来安装 [bash] rvmsudo yum install -y gcc-c++ patch readline readline-devel zlib zlib-devel libyaml-devel libffi-devel openssl-devel make bzip2 autoconf automake libtool bison iconv-devel [/bash] 到这里的话，RVM应该安装成功了，然后就是安装Ruby了，这里选择的是1.9.2（这里下载有可能会失败的，试多几次吧。） [bash] rvm install 1.9.2 [/bash] 一堆下载、解压、编译、安装后，应该会提示一些信息，建议看看吧，接着设置默认的版本。 [bash] rvm use 1.9.2 --default [/bash] 最后把最新的Ruby gems安装上 [bash] rvm rubygems current [/bash]

All done.
